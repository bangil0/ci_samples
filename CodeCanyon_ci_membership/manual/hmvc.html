HMVC modular extensions

What happened?

We now use Wiredesignz - codeigniter-modular-extensions-hmvc as a base to make the code modular. Each module now has its own controllers, modules and views where before these were all mixed general folders. You are not obligated to use HMVC, the normal Codeigniter way still works!

Which modules are included in this package?

Adminpanel module

For all the backend work we have just this one folder. Only put SETTINGS related code here. If you want to add new functionalities make a new module but you can centralize settings here in case it is more useful.

Auth module

Login, register functionality with options. Social login (OAuth) is also addressed here.

Home module

Static module to display the default page - the default module loaded = home as defined in routes.php: $route['default_controller'] = 'home';.

Membership module

Member accessible settings pages go here. Profile page and personal dashboard should go here.

Page layouts module

Is just a basic module for showing off the different page layouts that are possible with this product.

Utils module

Holds stuff that could be considered helpers but doesn't really belong anywhere. Could be split up and sanitized further in the future as things keep growing.

Controllers:
  • Install: entry point for the external installer to finish up.
  • Page_not_found: takes care of showing a custom page when something isn't there. 404 handler. Routes.php contains the configuration: $route['404_override'] = 'utils/page_not_found';.
  • Site_offline: shows an offline message
Modules:
  • Rbac_model: takes care of database interaction for roles and permissions. Since it's only one file making a separate module for it would be overkill.
  • Set_cookies_model: load session vars from cookie. Could be handy for cross-domain cookie sharing?
  • Settings_model: is autoloaded and settings are saved to cache file.
